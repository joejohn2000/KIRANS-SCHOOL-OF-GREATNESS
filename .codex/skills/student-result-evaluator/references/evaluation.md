# Evaluation Reference

## Post-Submission Checklist

1. Locate the correct dated folder.
1. Read the paper, teacher key, student answers, current plan, and current progress card.
1. Save the student's actual answers into `candidate-answers.md` if needed.
1. Score the work question by question.
1. Write `evaluation-score.md`.
1. Add `focus-session.md` only if the weakness pattern needs targeted follow-up.
1. Update `plan.md` if the next step changes.
1. Update `progress-card.md` if the long-term memory changes.

## What Counts As Durable Evidence

Good reasons to update `progress-card.md`:
- repeated implementation weakness across dates
- clear improvement in a previously weak topic
- better explanation quality under pressure
- new stable strength in tracing, complexity, or edge-case handling

Bad reasons to update it:
- one typo
- one isolated bad answer
- one lucky multiple-choice hit

## Plan Update Heuristics

- `<70%` or major blanks: do not fully advance
- `70-84%`: advance carefully with carry-over
- `85%+`: advance with light review

Also apply the repo rules:
- one failed gate -> carry-over next day
- two consecutive failed days -> catch-up day
- later adapted phases -> no blanks, explanation-first, complexity justification mandatory

## Recommended Language

- "The student understands the concept but loses marks on implementation precision."
- "This result supports advance with carry-over, not a clean advance."
- "The weakness is now repeated enough to update the progress memory."
- "The next day should be narrowed rather than widened."
